# 8大排序



# 最坏时间复杂度
快速排序
平均：O(n log n)
最坏：O(n^2)

归并排序
平均：O(n log n)
最坏：O(n log n)
但需要额外 O(n) 空间

堆排序
平均：O(n log n)
最坏：O(n log n)
只需要 O(1) 额外空间
但实际常数通常比快排大一些

# 选什么排序
数据需要稳定排序
常用归并排序，不一定用快排。

必须保证最坏 O(n log n)
常用堆排序、归并排序，或者 Introsort。

数据量小
插入排序可能更快，因为常数很小。

整数范围不大、满足特殊条件
计数排序、基数排序可能比比较排序更快，甚至能到 O(n)。

工程上常见做法：
小数组时切换插入排序
递归太深时切换堆排序
